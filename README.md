# LingoSnap
Instant translation at your fingertips

项目概述：
这是一个计划能够运行于ubuntu平台的翻译软件（后续有在windows，其他linux和mac上开发的计划，因此尽量选择可迁移的架构）
这个翻译软件的核心功能是通过两个便捷的快捷方式捕获文字，并直接跳出翻译窗口进行翻译达到差异化的目的。在ubuntu中还应该支持在terminal中的翻译。
两个快捷方式是： 1. 选择文字后按下预设好的快捷键启动翻译。 2. 按下预设好的快捷键以后启动截图功能，将用户截取到的内容进行ocr识别，将识别到的文字启动翻译
翻译计划使用两种引擎：1. Argos Translate[https://github.com/argosopentech/argos-translate]（使用本地部署的模型离线翻译） 2. Google Translate API[https://cloud.google.com/translate/docs] (调用远程API在线翻译,用户需要自行提供API key)
屏幕OCR计划使用开源模型：Tesseract OCR[https://github.com/tesseract-ocr/tesseract]

GUI的描述：
项目最终应该能够被打包成为一个安装包，如deb文件。运行后能够在ubuntu的桌面创建一个启动快捷键，启动后程序能够被启动，并且维持在系统托盘中（关闭GUI并不会结束程序，结束程序需要在托盘中右键程序选择结束）。GUI应该能够提供中文和英文两种显示语言。
GUI窗口应该由两部分内容组成：窗口上方的横条菜单栏（分别有：Text Translate， History和 Settings），窗口下方的部分则是对应的展示内容。
1. Text Translate的界面应该有选择初始语言与目标语言的下拉栏， 中间有一个交换初始语言和目标语言的按钮。 再下方是两个大的文本框，左边的的是输入待翻译文本的文本框，左边是已经翻译完成的文本框。每个文本框中的内容都应该是能够被编辑的，但只有左边的输入框被编辑后才会触发启动翻译。 启动翻译的逻辑是：如果是通过快捷方式键入的文本则立即翻译，如果是用户在文本框中打字，则检测到2秒没有键入文字以后开始翻译。 在等待得到翻译结果的时候应该有一个简单的加载动画。 文本框如果有太多文字，应该有滚动条，左右两侧的内容应该同步滚动。字体的大小应该让人舒适（太小影响观看，太大影响阅读效率）。
2. History界面应该存储每次翻译的记录（能够在设置中设置存储的数量），每条记录以卡片的形式展现：上方是翻译的时间，原始语言和目标语言；中间是原始内容摘要；下方是目标结果摘要。在点击了卡片以后，将会跳转至Text Translate界面，将原始内容放置在待翻译的文本框中，右侧是翻译后的内容。如果历史记录过多，应该在这个界面中添加滚动条
3. 设置内容应该包含软件的各项设置：
     1. 启动翻译的快捷键（默认为ctl+c+c，及按下ctl同时快速按下两次c，快捷键也要运行用户自己定制）
     2. 启动ocr的快捷键（默认为ctl+F8）.
     3. 选择使用的翻译引擎（为被选择的翻译引擎的对应设置也应该被展示，但是应该是灰色，即使切换引擎，另外一个引擎的设置也应该被保留）
       + Google Translate API被选择后的设置，即要求用户设置API key
       + Argos Translate 被选择后的设置：可能需要用户管理语言包，需要展示已经安装的语言包和Install Package的选项
     4. 翻译器的页面语言：英语/中文
     5. 关于，包含logo和链接到github仓库
